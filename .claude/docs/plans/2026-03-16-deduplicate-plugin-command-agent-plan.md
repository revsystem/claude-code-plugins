# プラグイン command/agent 間の重複解消プラン

作成日: 2026-03-16
関連 PR: #10
関連 Issue: #9

## 背景

terraform-code-reviewer プラグインの command と agent に、レビュープロセス、フィードバック形式、レビュー観点、注意事項が重複して定義されていた。command は agent が1つしかないにもかかわらず、agent の内容を要約して埋め込んでおり、オーケストレーションとしての固有の役割が薄かった。

document-reviewer プラグインは command + 2 agents の構成で、command が2つの agent の並行実行と結果統合というオーケストレーション責務を持っており、重複は最小限に抑えられている。この設計パターンを基準として、terraform-code-reviewer の command を改善した。

## 設計方針

command と agent の責務分離を以下のように定義する。

| 種別 | 責務 | 含めるべき内容 |
|------|------|----------------|
| command | ユーザー向けインターフェース | 使い方、引数、出力例、agent への委任関係 |
| agent | レビューロジック本体 | レビュープロセス、チェックリスト、フィードバック形式、詳細な観点 |

agent を廃止して skill(= command)に一本化する案も検討したが、以下の理由で agent は維持する。

- 自動ディスパッチ: ユーザーが自然言語で依頼するだけでシステムが agent を自動選択・起動できる
- コンテキスト隔離: レビュー作業がメインの会話コンテキストを消費しない
- ツール制限: agent は frontmatter の tools で使用可能ツールを制限できる

## 実施内容

### terraform-code-reviewer (実施済み)

command から以下の重複セクションを削除し、agent への委任に一本化した。

- レビュープロセス(フェーズ1-3)
- 統合フィードバック形式(🔴🟡🟢の詳細カテゴリ)
- レビュー観点(セキュリティ/ベストプラクティス/パフォーマンス/コスト最適化の詳細項目)
- 注意事項

command に残した内容:

- コマンド説明と agent への委任関係
- 使用方法(基本/ファイル指定/観点指定/レベル指定の4パターン)
- 出力例(サマリーと詳細フィードバックの項目)
- エージェント連携(agent の担当・対象・出力の概要)

結果: 145行 → 71行(51%削減)

### document-reviewer (変更不要)

command は 2 agent のオーケストレーション(並行実行 + 結果統合)という固有の責務が明確。統合フィードバック形式は各 agent の出力を横断的に分類した統合版であり、agent 側のフォーマットとは粒度が異なる。注意事項に若干の内容的重なりがあるが、command は全体品質の方針、各 agent は担当領域固有の注意事項で役割が分かれている。

### aws-cost-analyst, tech-docs-searcher (該当なし)

agent のみの構成で command がないため、重複は構造上発生しない。

## 今後の適用指針

新しいプラグインを作成する際、または既存プラグインに command を追加する際は、以下の基準に従う。

- command には agent のロジックを要約して埋め込まない
- command は「何ができるか」「どう使うか」「出力はどうなるか」に集中する
- レビューロジック、チェックリスト、フィードバック形式は agent に集約する
- 複数 agent を持つ場合のみ、command にオーケストレーション(並行実行、結果統合)のロジックを含める
