## 決定事項

- plugin 版は skills 版の frontmatter をそのまま使わず、既存決定（2026-05-25 ノート参照）に従い `disallowedTools: Write, Edit` 方式・エージェント名 `terraform-code-reviewer-agent`・プラグインレベル MCP 定義を維持した。本文と description と model のみ skills 版に揃えた
- 「指摘しない項目」第1項の文言を skills 版の「このエージェントは実行ツールを持たない」から「レビューは静的分析で行う」に調整した。理由: plugin 版は denylist 方式のため Bash を保持しており、原文のままでは事実と異なるため
- バージョンはパッチバンプ（0.1.2）。本文書き直し＋model 固定は refactor 扱いでインターフェース変更なしと判断

## 捨てた選択肢と理由

- skills 版 frontmatter（tools allowlist + インライン mcpServers）の丸ごと移植: プラグイン提供 MCP はツール名が `mcp__plugin_<plugin>_<server>__<tool>` とスコープされ、ベア名 allowlist の動作が未検証。プラグインは `mcpServers` フロントマターを使えない — 既存の `disallowedTools: Write, Edit` 方式を維持
- 「実行ツールを持たない」文言のそのまま移植: plugin 版エージェントは Bash を保持するため事実と異なる — 「レビューは静的分析で行う」に言い換え
- README 内のプラグインインストール UI 例にある `v0.0.4` 表記の更新: UI キャプチャの再現例であり実バージョン管理の対象外（前セッションでも据え置き）— 今回も変更せず

## 学び

- skills 版の本文を plugin に移植する際は、ツール構成の差（allowlist で Bash なし vs denylist で Bash あり）に依存する記述がないか確認が必要。今回は「実行ツールを持たない」という一文が該当した
