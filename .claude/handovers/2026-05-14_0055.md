## 捨てた選択肢と理由

- HashiCorp terraform-mcp-server (Docker): Docker が必要でユーザーフレンドリーでない — aws-documentation-mcp-server（uvx）に移行
- HashiCorp terraform-mcp-server (go install): Go バイナリのインストールが必要でユーザーフレンドリーでない — 同上
- MCP なし（モデルの知識のみ）: AWS 公式ドキュメント参照による根拠提示がレビュー品質上重要 — documentation MCP は維持する方針を採用
