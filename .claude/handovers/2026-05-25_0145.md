## 学び

- MCP ツールの正しい表記はダブルアンダースコア（`mcp__server` で全ツール、`mcp__server__tool` で個別、`mcp__server__*` ワイルドカード）。`mcp_server`（1本）は不正で、tools アロウリストでは MCP にマッチせずアクセス不能になる
- サブエージェントの tools をアロウリスト指定すると、MCP を明示列挙しない限り MCP ツールは使えない。`disallowedTools` デナイリストなら MCP・読み取り系を自動継承し Write/Edit のみブロックできる（公式 sub-agents ドキュメントの明記）
- プラグイン配下の MCP 実ツール名は `mcp__plugin_<plugin>_<server>__<tool>` とスコープされる（このセッションの deferred tool 一覧で確認）
- プラグインエージェントは `hooks`/`mcpServers`/`permissionMode` フロントマターを使えない（セキュリティ上無視される）。MCP サーバーは marketplace.json のプラグインレベル mcpServers で定義する
- 公式 plugins-reference のエージェント例自体が `disallowedTools: Write, Edit` を使っており、プラグイン read-only エージェントの推奨パターンと判断できる
