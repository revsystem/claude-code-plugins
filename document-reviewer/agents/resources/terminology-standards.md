# 用語規約

**一般的なルール:**

- 技術用語には適切な大文字小文字の使い分けを使用する
- 製品とサービスには公式の命名に従う
- 日本語のカタカナ用語には長音記号を使用する（ユーザー、サーバー）
- 英語の固有名詞は原語のままで表記する
- 技術用語には一般的に認知された日本語翻訳を使用する
- 引用文では元の表記を維持する
- **``` で囲まれたコードブロックは校正ルールの対象外** - ターミナルコマンドとソースコードはそのまま残す

**AI/ML 用語:**

- RAG (rag, Rag ではない) - Retrieval-Augmented Generation
- API (api, Api ではない) - Application Programming Interface
- JSON (json, Json ではない) - JavaScript Object Notation
- CSV (csv, Csv ではない) - Comma-Separated Values
- SaaS (Saas, saas ではない) - Software as a Service
- MCP (mcp ではない) - Model Context Protocol
- MCP サーバー (MCPサーバ, MCP Server ではない) - 英語と日本語の間にスペースを入れる
- Claude (claude, CLAUDE ではない) - Anthropic の AI モデル
- Claude デスクトップ (Claudeデスクトップ ではない) - 英語と日本語の間にスペースを入れる
- LLM (llm, Llm ではない) - Large Language Model
- Token（複数のトークンを指す場合以外は単数形を使用）
- Chunking (chunking ではない) - データ処理技術
- ベクター (ベクタ ではない) - vector、長音記号を使用
- エンベディング - embedding、AI/ML 用語
